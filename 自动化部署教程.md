# VSCode → GitHub → PythonAnywhere 自动化部署教程

## 🎯 目标

```
你修改代码 → VSCode 一键推送 → GitHub 保存代码 → PythonAnywhere 自动更新网站
```

---

## 📋 第一步：在 GitHub 上创建仓库

1. 打开 https://github.com/ ，登录你的 GitHub 账号
2. 点击右上角 **「+」** → **「New repository」**
3. 填写：
   - **Repository name**: `my-blog`（随便起名）
   - **Description**: `我的个人博客`
   - **Public**（公开，免费） 或 Private
   - ❌ **不要勾选** "Add a README file"（后面自己加）
   - ❌ **不要勾选** ".gitignore"（项目里已经有了）
4. 点击 **「Create repository」**
5. 创建后会看到一个页面，显示类似：
   ```
   git remote add origin https://github.com/你的用户名/my-blog.git
   git branch -M main
   git push -u origin main
   ```
   先不要执行，记住这个地址。

---

## 📋 第二步：在 VSCode 中推送到 GitHub

### 2.1 初始化 Git 仓库

打开 VSCode 的终端（Ctrl+`），执行：

```bash
cd ~/code/blog

# 初始化 Git 仓库（如果还没初始化）
git init

# 添加所有文件到暂存区
git add .

# 首次提交
git commit -m "🎉 初始提交：个人博客项目"
```

### 2.2 关联 GitHub 远程仓库

```bash
# 把 "你的用户名" 换成你实际的 GitHub 用户名
git remote add origin https://github.com/你的用户名/my-blog.git

# 主分支改名为 main
git branch -M main

# 推送到 GitHub
git push -u origin main
```

> 执行 `git push` 时会弹出一个窗口让你登录 GitHub。选择 **「Sign in with your browser」** 即可。

### 2.3 以后每次修改代码后，只需三步

```bash
git add .                          # 1. 暂存修改
git commit -m "描述你改了什么"        # 2. 提交到本地
git push                           # 3. 推送到 GitHub
```

**VSCode 图形化操作（更简单）：**
- 左侧栏点击 **「源代码管理」** 图标（分支图标）
- 在输入框写提交信息
- 点击 **「✓ 提交」**（或 Ctrl+Enter）
- 点击 **「⋯」** → **「推送」**

---

## 📋 第三步：在 PythonAnywhere 上克隆仓库

### 3.1 首次克隆

登录 PythonAnywhere，打开 **Bash** 终端：

```bash
# 克隆你的 GitHub 仓库到 PythonAnywhere
git clone https://github.com/你的用户名/my-blog.git ~/blog

# 进入项目目录
cd ~/blog

# 安装依赖
pip install --user -r requirements.txt
```

### 3.2 配置 Web 应用

在 **Web** 标签页，按照之前的教程配置 WSGI 文件（指向 `~/blog/wsgi.py`）。

---

## 📋 第四步：设置自动部署（核心！）

这里提供 **两种方案**，从简单到高级。

---

### 方案一：定时自动拉取（最简单，推荐初学者）

PythonAnywhere 免费版支持 **Scheduled Tasks**（定时任务），让它每隔一段时间自动从 GitHub 拉取最新代码。

**操作步骤：**

1. 在 PythonAnywhere 顶部点击 **「Tasks」** 标签
2. 在 **「Schedule a task」** 输入框中输入：
   ```
   cd ~/blog && git pull
   ```
3. 选择频率：**Hourly**（每小时检查一次）或 **Daily**（每天一次）
4. 点击 **「Create」**

```
┌──────────┐    push    ┌──────────┐   定时 pull   ┌──────────────────┐
│  VSCode  │ ────────→  │  GitHub  │ ──────────→  │ PythonAnywhere   │
│ 写代码    │  手动触发   │ 代码仓库  │  自动（定时）  │ 你的网站 ✨       │
└──────────┘            └──────────┘               └──────────────────┘
        每次 push 后最多等 1 小时网站就会自动更新
```

> ⚠️ 定时任务执行 `git pull` 后，PythonAnywhere **不会自动重载** Web 应用。
> 你需要额外做一步：点击 **「Web」** 标签 → **「Reload」**。
>
> **自动化 reload 技巧：** 把定时任务改成：
> ```
> cd ~/blog && git pull && touch /var/www/你的用户名_pythonanywhere_com_wsgi.py
> ```
> `touch` 命令会"摸一下" WSGI 配置文件，PythonAnywhere 检测到文件变化就会自动重载！

---

### 方案二：GitHub Webhook 即时部署（更快！）

如果你不想等定时任务，可以设置 **Webhook**。GitHub 会在你每次 push 时，立刻通知 PythonAnywhere 拉取最新代码。

#### 4.1 创建 Webhook 接收脚本

在 PythonAnywhere 的 Bash 终端中：

```bash
# 创建 ~/webhook_app 目录
mkdir ~/webhook_app

# 创建 webhook 接收服务
cat > ~/webhook_app/app.py << 'PYEOF'
"""
一个独立的微型 Web 服务，专门接收 GitHub 的 Webhook 通知。
收到通知后，自动 git pull 并触发网站重载。

这不是你的博客！这是部署助手。
"""
import os
import hmac
import hashlib
import subprocess
from flask import Flask, request

app = Flask(__name__)

# ⚠️ 设置一个密钥（自己随便编一段字符串），防止别人乱触发你的部署
SECRET = b"my-super-secret-blog-2026"

# PythonAnywhere 上的项目路径和 WSGI 文件路径
# 把 "你的用户名" 改成你的实际用户名！
PROJECT_PATH = "/home/你的用户名/blog"
WSGI_FILE = "/var/www/你的用户名_pythonanywhere_com_wsgi.py"


@app.route("/webhook", methods=["POST"])
def webhook():
    # 验证请求来自 GitHub（防止恶意请求）
    signature = request.headers.get("X-Hub-Signature-256", "")
    expected = "sha256=" + hmac.new(SECRET, request.data, hashlib.sha256).hexdigest()
    if not hmac.compare_digest(signature, expected):
        return "Invalid signature", 403

    # 执行 git pull
    result = subprocess.run(
        ["git", "-C", PROJECT_PATH, "pull"],
        capture_output=True, text=True
    )
    print("Git pull:", result.stdout, result.stderr)

    # touch WSGI 文件，触发网站重载
    if os.path.exists(WSGI_FILE):
        os.utime(WSGI_FILE)
        print("WSGI file touched, app will reload.")

    return "OK", 200
PYEOF

echo "✅ webhook_app/app.py 创建完成"
```

然后安装 Flask（轻量级 Web 框架，比 FastAPI 更简单）：

```bash
pip install --user flask
```

#### 4.2 在 PythonAnywhere 上创建第二个 Web 应用

PythonAnywhere 免费版只能有 **1 个** Web 应用…

> **这就是免费版的限制。** 方案二在免费版上行不通，因为你已经有了博客这个 Web 应用。

**解法：把 Webhook 路由直接加到你的博客里！**

在 [`main.py`](blog/main.py) 中添加一个 `/webhook` 路由：

在 PythonAnywhere 上编辑 `~/blog/main.py`，在文件末尾（`delete_post` 之后）添加：

```python
# ============================================================
# Webhook 路由 - GitHub 通知自动部署
# ============================================================
import hashlib
import hmac
import subprocess

WEBHOOK_SECRET = b"my-super-secret-blog-2026"  # 自己设个密码


@app.post("/webhook")
async def github_webhook(request: Request):
    """
    接收 GitHub 的 Webhook 推送通知
    GitHub 在你每次 git push 后会自动调用这个地址
    """
    # 1. 验证签名（确认请求确实来自 GitHub）
    body = await request.body()
    signature = request.headers.get("X-Hub-Signature-256", "")
    expected = "sha256=" + hmac.new(
        WEBHOOK_SECRET, body, hashlib.sha256
    ).hexdigest()
    if not hmac.compare_digest(signature, expected):
        return {"error": "无效签名"}, 403

    # 2. 执行 git pull
    import os
    result = subprocess.run(
        ["git", "-C", str(BASE_DIR), "pull"],
        capture_output=True, text=True,
    )
    print("🔁 git pull:", result.stdout or result.stderr)

    # 3. touch WSGI 文件，触发 PythonAnywhere 重载
    wsgi_file = os.environ.get("WSGI_FILE", "")
    if wsgi_file and os.path.exists(wsgi_file):
        os.utime(wsgi_file)
        print("✅ WSGI 文件已更新，网站将自动重载")

    return {"status": "ok", "output": result.stdout or result.stderr}
```

> ⚠️ 如果你选择方案一（定时任务），就不需要添加这个路由。保持代码简洁。

#### 4.3 在 GitHub 上设置 Webhook

1. 打开你的 GitHub 仓库页面
2. 点击 **「Settings」** → **「Webhooks」** → **「Add webhook」**
3. 填写：
   - **Payload URL**: `https://你的用户名.pythonanywhere.com/webhook`
   - **Content type**: `application/json`
   - **Secret**: `my-super-secret-blog-2026`（和代码里的一致）
   - **Which events**: 选 **「Just the push event」**
4. 点击 **「Add webhook」**

3. 等定时任务跑完（或手动去 PythonAnywhere 点 Reload）

## ❓ 常见问题

### git push 失败：`Permission denied`

说明没有登录 GitHub。在 VSCode 终端中执行：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

然后再次 push，按弹窗提示登录。

### 为什么不把 blog.db 上传到 GitHub？

`blog.db` 是你的文章数据，包含个人内容。而且不同电脑/服务器上的数据库路径不同，上传会出问题。`.gitignore` 已经帮你排除了。

### 免费版 PythonAnywhere 能自动 deploy 吗？

能，但有限制：
- ✅ 可以设置定时任务 `git pull`
- ✅ 可以 `touch` WSGI 文件触发重载
- ⚠️ 免费版只能有 1 个 Web 应用（所以方案二的第二个 Web 应用不可行）
- ⚠️ 最快是每小时更新一次（定时任务最小间隔）
